## 二、理解维度模块（正在建设 🚧）

这些模块共享同一套基础设施（记忆写入/搜索/投影），区别只在于 **event type 分类** 和 **前端展示方式**。

### 2.1 基础画像（Basic Profile）
- **事件类型**：`profile_updated`
- **内容**：名字、简介、当前阶段
- **状态**：字段仍含求职残留（school_name/gpa），需清理
- **优先级**：P0

### 2.2 兴趣与价值观（Interests & Values）
- **事件类型**：`interest_observed`、`value_surfaced`
- **内容**：用户真正着迷的事、在意什么、底线是什么
- **状态**：✅ 后端完整，Agent 工具可用

### 2.3 情绪规律（Emotional Patterns）
- **事件类型**：`emotional_pattern`
- **内容**：反复出现的感受模式
- **状态**：✅ 后端完整，缺可视化

### 2.4 重要经历（Significant Moments）
- **事件类型**：`significant_moment`
- **内容**：对用户有意义的事
- **状态**：✅ 后端完整，Journey 时间线已展示

### 2.5 重要关系（Relationships）
- **事件类型**：`relationship_noted`
- **内容**：谁占据重要位置
- **状态**：✅ 后端完整，前端未展示

### 2.6 重要决策（Decisions）
- **事件类型**：`decision_made`
- **内容**：值得记录的选择
- **状态**：✅ 后端完整，前端未展示

### 2.7 自我反思（Reflections）
- **事件类型**：`reflection_added`
- **内容**：对话洞察、用户说过的关键话
- **状态**：✅ 后端完整，前端未展示

### 2.8 矛盾观察（Contradictions）
- **事件类型**：`contradiction_noted`
- **内容**：说 X 做 Y 的观察
- **状态**：✅ 后端完整，Agent 工具可用，从未触发过（最难自动识别）

## 三、扩展模块（待开发 📋）

### 3.1 模式洞察（Pattern Insight）
- **定位**：AI 从大量事件中提炼出的跨维度模式
- **示例**："你每次面临选择前都会焦虑 2-3 天"、"你提到妈妈时总是用敬语"
- **技术**：LLM 聚合分析 + 定期刷新
- **工作量**：1 天
- **优先级**：P0 — 当前永远为空，体验断层

### 3.2 主动建议（Proactive Suggestion）
- **定位**：不是等用户问，而是在合适的时机主动给出建议
- **触发条件**：
  - 用户反复表达同一类困惑 → 主动总结并建议
  - 检测到情绪低谷 → 主动询问或提供支持
  - 用户之前设定的意图被遗忘 → 温和提醒
- **技术**：定期扫描画像变化 + LLM 判断时机
- **工作量**：2-3 天
- **优先级**：P1

### 3.3 意图/目标追踪（Intent Tracking）
- **定位**：用户说过"我想..."、"我希望..."，Lumen 记住并追踪进展
- **示例**：
  - 用户说"我想学日语" → Lumen 记录，一个月后问"日语学得怎么样了？"
  - 用户说"我想多出门走走" → 检测到连续 7 天没记录出门事件 → 温和提醒
- **事件类型**：复用 `value_surfaced` 或新增 `intent_declared`
- **工作量**：1-2 天
- **优先级**：P1

### 3.4 情绪追踪可视化（Mood Timeline）
- **定位**：时间维度上的情绪起伏图
- **技术**：从 `emotional_pattern` 事件中提取情绪标记，生成折线图/热力图
- **前端**：Profile 页面新增"情绪"标签页
- **工作量**：1 天
- **优先级**：P1

### 3.5 关系网络图（Relationship Graph）
- **定位**：可视化用户提到的重要人物及其关系动态
- **技术**：从 `relationship_noted` + `significant_moment` 中提取人物关系
- **前端**：力导向图或简单列表时间线
- **工作量**：2 天
- **优先级**：P2

### 3.6 外部数据源接入（Data Sources）
- **定位**：自动从 Obsidian、Notion、日历等导入数据，丰富理解维度
- **已有基础**：DataSourceConnector 抽象 + IngestionPipeline
- **优先级**：P2 — 基础设施已有，缺具体连接器实现

### 3.7 对话回顾（Conversation Retrospective）
- **定位**：每周/每月生成一段"这段时间我们聊了什么、我发现了什么"的回顾
- **技术**：聚合近期事件 + LLM 生成自然语言回顾
- **前端**：Profile 页面新增"回顾"标签页，或推送到对话中
- **工作量**：1 天
- **优先级**：P2

### 3.8 梦境/日记记录（Dream & Journal）
- **定位**：用户主动记录的非对话内容（梦境、每日日记）
- **技术**：新增 event type `journal_entry`，支持时间戳和标签
- **前端**：简单的文本输入框 + 时间线展示
- **工作量**：半天
- **优先级**：P3
